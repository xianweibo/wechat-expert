# 接收代码审查

## 核心原则
先验证再实施。先提问再假设。技术正确性优先于社交舒适度。

## 响应模式

1. 阅读：完整阅读反馈，不急于反应
2. 理解：用自己的话复述需求（或提问）
3. 验证：对照代码库实际情况检查
4. 评估：对这个代码库来说技术上合理吗？
5. 回应：技术性确认或有理有据的反驳
6. 实施：一次一项，逐个测试

## 禁止的回应
- "你说得太对了！" / "好观点！"
- "让我立刻实施"（在验证之前）

## 应该这样做
- 复述技术需求
- 提出澄清性问题
- 如果审查意见有误，用技术理由反驳
- 直接动手做（行动胜于言辞）
