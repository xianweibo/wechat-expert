# 中文代码审查规范

## 核心原则
用"建议"代替"命令"，用"提问"代替"否定"，但绝不因为面子而放过 bug。

## 分级标注
- **[必须修复]** — 安全漏洞、数据丢失风险、逻辑错误
- **[建议修改]** — 性能问题、可维护性、缺少校验
- **[仅供参考]** — 命名优化、风格建议、替代方案
- **[问题]** — 不确定的地方，需要作者解释意图

## 表达方式
| 避免 | 推荐 |
|------|------|
| 你必须改成 X | 建议考虑用 X，因为 Y |
| 这里写错了 | 这里可能存在一个问题... |
| 不要用这个方法 | 这个方法在 A 场景下可能有性能问题 |

## 中英混排规范
- 业务逻辑注释用中文
- 变量/函数名始终英文
- 中英文之间加空格
- 技术术语保留英文不翻译
