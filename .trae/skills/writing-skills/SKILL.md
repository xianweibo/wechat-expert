# 编写技能

## 核心原则
编写技能就是将测试驱动开发应用于流程文档。
没有失败的测试就不写技能。

## SKILL.md 结构
```markdown
---
name: skill-name-with-hyphens
description: Use when [具体的触发条件和症状]
---

# 技能名称
## 概述
## 何时使用
## 核心模式
## 快速参考
## 常见错误
```

## 关键要求
- description 以"Use when..."开头，只描述触发条件
- 使用第三人称
- 保持简洁（<500 词）
- 一个优秀的示例胜过多个平庸的

## 测试流程
1. 红：在没有技能的情况下运行压力场景，记录基线行为
2. 绿：编写针对具体合理化借口的技能
3. 重构：堵住漏洞，重新测试
