# 并行分派智能体

每个独立问题域分派一个专用智能体，让它们并发工作。

## 适用场景
- 3 个以上测试文件因不同根因失败
- 多个子系统独立出现故障
- 排查之间无共享状态

## 不适用场景
- 失败是相关的（修复一个可能修复其他的）
- 需要理解完整系统状态
- 智能体之间会互相干扰

## 流程
1. 识别独立的问题域
2. 创建聚焦的智能体任务（明确范围、清晰目标）
3. 并行分派
4. 审查与集成结果
